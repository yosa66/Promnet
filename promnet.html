Pengenalan Git

Git adalah sistem kontrol versi yang populer digunakan oleh pengembang perangkat lunak untuk mengelola perubahan dalam kode sumber proyek. Ini adalah alat yang sangat berguna untuk kolaborasi tim, melacak perubahan, dan mengelola riwayat proyek.

Repositori Git

Repositori Git adalah tempat penyimpanan untuk semua perubahan kode sumber proyek. Repositori ini dapat berada di komputer lokal Anda (repositori lokal) atau di server jarak jauh (repositori jarak jauh).

Beberapa perintah Git yang berguna:

- git init: Memulai repositori lokal baru.
- git clone [URL]: Mengambil repositori jarak jauh ke komputer lokal.

Commit

Commit adalah tindakan menyimpan perubahan tertentu dalam repositori Git. Setiap commit memiliki pesan yang menjelaskan perubahan yang dilakukan. Ini membantu dalam melacak dan mengelola perubahan.

Beberapa perintah Git yang berguna:

- git add [file]: Menambahkan perubahan ke staging area.
- git commit -m "[pesan]": Membuat commit dengan pesan deskripsi.

Cabang (Branch)

Cabang adalah versi terpisah dari repositori utama. Pengembang dapat membuat cabang untuk bekerja pada fitur atau perbaikan tertentu tanpa memengaruhi kode di repositori utama.

Beberapa perintah Git yang berguna:

- git branch: Menampilkan daftar cabang.
- git checkout [nama-cabang]: Beralih ke cabang yang ada.
- git merge [nama-cabang]: Menggabungkan perubahan dari cabang ke cabang saat ini.

Pull Request

Pull request adalah permintaan untuk menggabungkan perubahan dari sebuah cabang ke repositori utama. Ini sering digunakan dalam kolaborasi tim untuk meninjau, mengomentari, dan menguji perubahan sebelum penggabungan. Beberapa platform hosting Git, seperti GitHub dan GitLab, menyediakan fitur pull request yang kuat.

Git Command Line

Git dapat digunakan melalui command line (CLI) dengan perintah seperti git clone, git add, git commit, git push, dan lainnya. Ini adalah cara kuat untuk mengelola Git. Pastikan Anda menginstal Git pada komputer Anda dan mengonfigurasi nama pengguna dan email Anda dengan perintah git config.
